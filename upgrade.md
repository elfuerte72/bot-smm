# upgrade.md — идеи развития бота через Anthropic SDK

Список возможностей, которые можно добавить в бота, используя Anthropic Python SDK.
Отсортировано по приоритету: от «чинит текущую боль» к «новые фичи».
Для каждого пункта указаны затрагиваемые файлы и эскиз реализации.

Контекст бота см. в `CLAUDE.md`. Ключевые файлы:
- `src/agent/news_agent.py` — вызовы Claude, парсинг, валидация
- `src/agent/prompts.py` — `SYSTEM_PROMPT`, `build_user_prompt`
- `src/agent/schemas.py` — `PostDraft` (pydantic)
- `src/storage/repo.py` — слой доступа к SQLite
- `src/media/og_image.py` — извлечение OG-картинки

---

## Уровень 1 — убирает то, что уже ломалось

### 1.1. Structured Outputs через `tool_use`

**Проблема.** Сейчас модель возвращает голый текст, JSON выколупывается регэкспом
`_extract_json`, а при провале валидации запускается хрупкий `_shrink_draft`
(именно он давал плавающий сбой «Не нашёл JSON в ответе модели на ретрае»).

**Решение.** Объявить Claude инструмент с `input_schema`, равной структуре
`PostDraft`, и форсировать его вызов (`tool_choice`). Модель обязана вернуть
валидную структуру — парсить нечего.

**Эффект.** Удаляются `_extract_json`, regex `_JSON_BLOCK_RE`, весь путь
`_shrink_draft`. `news_agent.py` заметно упрощается, класс багов с парсингом
исчезает.

**Файлы.** `src/agent/news_agent.py` (основной рефакторинг), `src/agent/schemas.py`
(схему `PostDraft` можно отдать как `input_schema` почти напрямую).

**Нюанс.** `web_search` — server tool, он работает в той же сессии; нужно
аккуратно совместить server tool (`web_search`) и наш client tool для вывода.
Вариант: первый вызов с `web_search`, второй вызов (без поиска) с форсированным
структурным инструментом.

### 1.2. `web_fetch` server tool для режима source_url

**Проблема.** В режиме `source_url` (`build_user_prompt`) мы просим модель
«открой эту страницу через web_search по точному URL» — ненадёжно, модель может
не открыть именно нужную страницу.

**Решение.** Использовать server tool `web_fetch` — он читает конкретный URL
напрямую.

**Файлы.** `src/agent/news_agent.py` (добавить tool), `src/agent/prompts.py`
(ветка `source_url` в `build_user_prompt` — упростить инструкции).

### 1.3. Citations

**Идея.** Anthropic умеет привязывать утверждения к фрагментам источника. Для
новостного бота это снижает риск выдуманных фактов.

**Эффект.** В превью админам можно показывать, какой факт из какого источника.

**Файлы.** `src/agent/news_agent.py`, `src/bot/handlers.py` (`_do_send_preview` —
опционально вывод цитат), возможно `src/agent/schemas.py`.

## Уровень 2 — качество контента

### 2.1. Self-review вторым вызовом («редактор»)

Отдельный дешёвый вызов Claude (без `web_search`): проверяет черновик против
антибот-фильтров и стиля из `SYSTEM_PROMPT`, возвращает исправленный вариант.
Точнее, чем детерминированный `_strip_dashes`.

**Файлы.** `src/agent/news_agent.py` (новый шаг после генерации).

### 2.2. Extended thinking для блока takeaway

Включить «размышление» модели, чтобы экспертный вывод (`takeaway`) был глубже
и неочевиднее. Стоит дороже — взвесить по бюджету (`api_usage`).

**Файлы.** `src/agent/news_agent.py`.

### 2.3. Vision — проверка OG-картинки

Прогнать найденную OG-картинку через Claude (image input): релевантна ли она
посту, не логотип/мусор ли. Сейчас `og_image.py` берёт картинку вслепую.

**Файлы.** `src/media/og_image.py`, `src/agent/news_agent.py` или
`src/bot/handlers.py` (`_persist_draft`).

## Уровень 3 — новые фичи

### 3.1. Реальный антидубль через custom tool + SQLite

**Проблема.** Сейчас антидубль «мягкий»: список тем (`repo.recent_topics()`)
просто передаётся в промпт, модель может его проигнорировать (был дубль
«Google Spark»).

**Решение.** Дать Claude client tool `check_topic_covered`, который ходит в нашу
SQLite через `repo.py`. Жёсткая проверка во время генерации.

**Файлы.** `src/storage/repo.py` (запрос проверки темы), `src/agent/news_agent.py`
(объявить tool и обработать его вызовы в цикле).

### 3.2. Batch API

Для cron генерировать пачкой со скидкой ~50%, либо делать несколько кандидатов
и выбирать лучший отдельным вызовом.

**Файлы.** `src/agent/news_agent.py`, `src/scheduler.py`.

### 3.3. Еженедельный дайджест

Раз в неделю собирать опубликованные посты (`repo`, таблица `published`) в один
обзорный пост. Новый формат контента.

**Файлы.** `src/scheduler.py` (новый job), `src/agent/` (новый промпт/функция),
`src/bot/handlers.py` (команда).

### 3.4. Режим перевода зарубежных новостей

Отдельная ветка генерации: взять зарубежную статью и переписать в стиле канала
на русском.

**Файлы.** `src/agent/prompts.py` (`build_user_prompt`), `src/bot/handlers.py`
(пункт меню).

### 3.5. Streaming прогресса в Telegram

Стримить ответ модели и редактировать статус-сообщение («Ищу инфоповод…»),
чтобы админ видел прогресс, а не ждал молча 30–60 сек.

**Файлы.** `src/agent/news_agent.py` (streaming-вызов), `src/bot/handlers.py`
(`_run_generation` — обновление статус-сообщения).

### 3.6. Files API / Memory tool

Загрузить эталонные посты и бренд-гайд как референс — стабильнее стиль без
раздувания `SYSTEM_PROMPT`.

**Файлы.** `src/agent/news_agent.py`, `src/agent/prompts.py`.

## Рекомендованный порядок внедрения

1. **Structured Outputs (1.1)** — чинит реальный класс багов, упрощает
   `news_agent.py`. Делать первым.
2. **Антидубль через tool + SQLite (3.1)** — закрывает известную проблему
   с повторами тем.
3. **`web_fetch` (1.2)** — быстрый выигрыш для режима «по ссылке».

Дальше — по желанию: self-review, citations, дайджест.

## Замечания

- `SYSTEM_PROMPT` помечен `cache_control` (кеш 5 мин). Любые новые
  tool-определения и system-блоки добавлять так, чтобы не ломать
  кеш-breakpoint (см. CLAUDE.md, раздел «Anthropic API»).
- Все новые вызовы должны писать расход в `api_usage` через
  `repo.record_api_usage` (см. `_log_and_record_usage`), иначе `/status` врёт.
- Сверяться с актуальной докой SDK при реализации — в проекте есть skill
  `claude-api` и MCP context7 для свежих доков.
